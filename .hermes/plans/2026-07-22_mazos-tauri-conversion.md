# MazOS → Tauri Desktop App Conversion Plan

> **For Hermes:** Use `subagent-driven-development` to implement task-by-task. Each task is one commit.

**Goal:** Convert MazOS from a Vercel-hosted Next.js dashboard (proxying to a local HTTP bridge) into a native Windows desktop app via Tauri v2, while keeping the same codebase deployable to Vercel as a hosted fallback. Add a minimalist "Toolkit" panel showing the most-used Hermes skills/MCP servers, and a CLI-inspired stats strip (context %, session cost, MCP status).

**Architecture:** Tauri v2 wraps the existing Next.js app. Next.js runs in **static export mode** (`output: 'export'`) for the desktop build — Tauri serves the static bundle from its own webview, no Node server needed on the client machine. All the data currently fetched over `http://127.0.0.1:3047` (the old bridge) is now fetched via Tauri's Rust-side commands (`invoke()`), which read the filesystem/git/processes directly — faster, no CORS, no bridge process to keep alive. The **Vercel-hosted build stays 100% unchanged** — it still runs `next build` (server mode) and keeps its own API routes; Tauri is an *additional* build target, not a replacement of the web one.

**Tech Stack:** Next.js 16 (existing) + Tauri v2 (Rust) + `@tauri-apps/api` (JS bridge) + existing Tailwind v4/Zustand. No new frontend framework.

**Design direction (per Maz: simple, minimalist, not AI-sloppy):**
- No glassmorphism, no gradient blur cards, no rounded-everything, no emoji-as-icons.
- Flat panels, 1px borders, monospace for data/stats (matches CLI feel), one accent color only.
- Reuse existing `panel` CSS class conventions from `page.tsx` — don't introduce a new design system on top of the existing one.

Note: Next.js doesn't natively support `NEXT_CONFIG_FILE` — simplest working approach is to **temporarily swap** `next.config.js` during the desktop build. Use a small Node script instead:

```js
// scripts/build-desktop.mjs
import { renameSync, copyFileSync, existsSync } from 'node:fs';
import { execSync } from 'node:child_process';

const orig = 'next.config.js';
const desktop = 'next.config.desktop.mjs';
const backup = 'next.config.js.bak';

renameSync(orig, backup);
copyFileSync(desktop, orig.replace('.js', '.mjs'));
try {
  execSync('next build', { stdio: 'inherit' });
} finally {
  renameSync(backup, orig);
}
```

Update script: `"build:desktop": "node scripts/build-desktop.mjs"`.

**Step 1: Add Rust command to read git status of tracked repos**
```rust
// src-tauri/src/commands.rs
use std::process::Command;

#[tauri::command]
pub fn git_status(repo_path: String) -> Result<String, String> {
    let output = Command::new("git")
        .args(["status", "--porcelain"])
        .current_dir(&repo_path)
        .output()
        .map_err(|e| e.to_string())?;
    Ok(String::from_utf8_lossy(&output.stdout).to_string())
}

#[tauri::command]
pub fn git_log_recent(repo_path: String, since: String) -> Result<String, String> {
    let output = Command::new("git")
        .args(["log", &format!("--since={}", since), "--pretty=format:%H|%s|%ad", "--date=iso"])
        .current_dir(&repo_path)
        .output()
        .map_err(|e| e.to_string())?;
    Ok(String::from_utf8_lossy(&output.stdout).to_string())
}
```

**Step 2: Register commands in main.rs**
```rust
// src-tauri/src/main.rs
mod commands;

fn main() {
    tauri::Builder::default()
        .invoke_handler(tauri::generate_handler![
            commands::git_status,
            commands::git_log_recent,
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

**Step 3: Create the JS-side abstraction (works in both Tauri and browser)**
```ts
// src/lib/mazos/bridgeClient.ts
export function isTauri(): boolean {
  return typeof window !== 'undefined' && '__TAURI__' in window;
}

export async function gitStatus(repoPath: string): Promise<string> {
  if (isTauri()) {
    const { invoke } = await import('@tauri-apps/api/core');
    return invoke('git_status', { repoPath });
  }
  // Fallback: existing HTTP bridge/API route for browser/Vercel
  const res = await fetch(`/api/mazos/git-status?repo=${encodeURIComponent(repoPath)}`);
  return res.text();
}
```

**Step 2: API route reads Hermes profile skill list + MCP config (server-side, Vercel-compatible)**
```ts
// src/app/api/mazos/toolkit/route.ts
import { NextResponse } from 'next/server';
import { readFileSync, existsSync } from 'node:fs';
import { join } from 'node:path';

const HERMES_PROFILE = process.env.HERMES_PROFILE_PATH || 'C:/Users/manaz/AppData/Local/hermes/profiles/maz-lite';

export async function GET() {
  // Skill usage requires a counter Hermes doesn't expose by default —
  // ship with a static curated "most useful" list first, real usage tracking is a follow-up loop.
  const topSkills = [
    { name: 'plan', category: 'software-development', useCount: 0 },
    { name: 'mazos-dashboard', category: 'software-development', useCount: 0 },
    { name: 'github-pr-workflow', category: 'github', useCount: 0 },
    { name: 'firecrawl-scraper', category: 'software-development', useCount: 0 },
    { name: 'financial-model-builder', category: 'productivity', useCount: 0 },
  ];

  const mcpConfigPath = join(HERMES_PROFILE, 'mcp.json');
  let mcpServers: { name: string; connected: boolean; tools: number }[] = [];
  if (existsSync(mcpConfigPath)) {
    try {
      const cfg = JSON.parse(readFileSync(mcpConfigPath, 'utf-8'));
      mcpServers = Object.keys(cfg.servers || {}).map((name) => ({
        name, connected: true, tools: (cfg.servers[name].tools || []).length,
      }));
    } catch { /* leave empty on parse error */ }
  }

  return NextResponse.json({ topSkills, mcpServers });
}
```


**Step 2: Create the new workflow**
```yaml
# .github/workflows/tauri-build.yml
name: Build Windows Desktop App

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch: {}

jobs:
  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
      - name: Install Rust
        uses: dtolnay/rust-action@stable
      - name: Install dependencies
        run: npm ci
      - name: Build Tauri app
        run: npm run tauri:build
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: mazos-windows-installer
          path: src-tauri/target/release/bundle/nsis/*.exe
      - name: Attach to release
        if: startsWith(github.ref, 'refs/tags/v')
        uses: softprops/action-gh-release@v2
        with:
          files: src-tauri/target/release/bundle/nsis/*.exe
```
